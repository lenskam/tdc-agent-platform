# Phase 4: Engineering & Automation

## Goal Description

Deploy the "Developer Agent" and "DevOps Agent" to accelerate software delivery.
The Developer Agent will handle boilerplate code generation and initial PR reviews.
The DevOps Agent will monitor system health and assist with deployment troubleshooting.

## User Review Required

> [!CAUTION]
>
> - **Production Access**: The DevOps agent should INITALLY have read-only access to production logs. Write access (restarting services) requires human approval.
> - **Code Interaction**: The Developer Agent will need a sandboxed environment to run generated code before committing.

## Verification Plan

### Automated Tests

- **Unit**: `pytest tests/tools/test_git_tools.py` (Mock git commands).
- **Integration**: Dev Agent -> Writes a `hello_world.py` -> Sandbox runs it -> Returns output.

### Manual Verification

- Ask Dev Agent: "Create a FastAPI endpoint for 'GET /users'".
- Verify it creates a file in `src/api/routes` (in a new branch).
- Ask DevOps Agent: "Why is the API slow?" (Mock a slow log entry).
- Verify it analyzes the provided log snippet.
